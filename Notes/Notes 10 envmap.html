Environment Maps
Motivation
  • Consider a metallic object
    • Most of the “metal” appearance comes from reflections
    • Metals have very little diffuse reflectivity; all specular
Motivation
  • Notice how the metallic appearance is much more convincing when reflections are present
  • Motivating demo: link
Motivation
  • Consider the case of shiny objects: They reflect things around them
  • Consider outdoor environments: We want to render the sky
  • Surprisingly, both of these can be addressed with the same mechanism!
Example
  • Suppose we have a 360° view of an environment
  • Imagine wrapping the environment image around a sphere
Example
  • Now, suppose we render a shiny object
  • Imagine we surround the object with the sphere
Procedure
  • For each pixel of the rendered object:
    • Compute the ray from the viewer to that point
      • Easy: We already have all the data we need
    • Compute the reflection ray
      • GLSL intrinsic: reflect( incoming , normal ) → outgoing
    • See where reflected ray hits sphere
    • Use that point’s color for the pixel in question
GPU
  • GPU provides some support for this operation
  • But: Spheres are a bit difficult to work with
  • So GPU pretends that we surrounded the object with a box instead
  • Six sides to a box
    • Left, right, top, bottom, front, back
Procedure
  • This is what GPU does internally:
    • Consider reflection vector
    • Find which component (x,y,z) has largest magnitude
      • If X:
        • If X<0 → use LEFT side
        • Else → use RIGHT side
      • If Y:
        • If Y<0 → use BOTTOM side
        • Else → use TOP side
      • If Z:
        • If Z<0 → use BACK side
        • Else → use FRONT side
Procedure
  • GPU uses other two coordinates to tell where on the face to sample
  • Ex: If reflection vector was (0.2,0.4,-0.8):
    • Largest magnitude is Z. Use BACK face
    • Other two components are x=0.2, y=0.4
    • Use x component to select texel column; y to select texel row
  • Ex: If reflection vector was (0.7,0.0,0.1):
    • Largest component is X. Use RIGHT face.
    • Other two components are y=0, z=0
    • Use z to select column; y to select row
GPU
  • New type of texture: cube map
    • Collection of six images
    • GL will sample from correct face (we’ll see how to do this in a moment)

Code
  • The framework contains a class that handles cubemap textures
    • ImageTextureCube.h, ImageTextureCube.cpp
  • We just need to use it
Setup
  • Loading a cubemap:
auto envMap = new ImageTextureCube( "skybox-%d.jpg" );
  • The %d will be substituted with 0,1,2,3,4,5
  • Each image must be square and the same size as the others
Draw
  • We need to bind the cubemap to one of the texture units
  • So our FS will have declarations like so:
layout(binding=0) uniform sampler2DArray baseTexture;
layout(binding=1) uniform sampler2DArray emissionTexture;
layout(binding=2) uniform sampler2DArray metallicRoughnessTexture;
layout(binding=3) uniform sampler2DArray bumpmapTexture;
layout(binding=4) uniform samplerCube envMap;      //NEW
Draw
  • Usually, the cubemap isn’t associated with a single mesh
  • Global to every object in the scene
  • So we’d likely not set it in Mesh::draw
  • Instead, set it in main draw():
envMap->bind(4);
FS
  • Now for our FS: Compute reflection vector and sample environment map:
void main(){
    ...
    vec3 V = normalize(eyePos - v_worldPos);
    vec3 VR = reflect(-V,N);
    ...compute color as usual...
    vec3 reflColor = texture(envMap, VR).rgb;
    color.rgb += metallicity * reflColor.rgb;
}
  • The more metallic an object is, the more reflective it is
  • Remember, metallicity comes from the metallicRoughness texture
Flaw
  • This scheme is pretty easy to implement
  • Fast to compute too
  • TANSTAAFL
  • So what’s the problem?
Flaw
  • Consider this situation: Viewer is moving perpendicular to a shiny wall
  • Observe that the reflection vectors are all parallel but they start from different places
  • So they might hit different things in the scene
Flaw
  • But remember how our environment mapping works
    • We only look at the reflection vector’s direction – not its origin
    • So we must get the same color reflected in each case
  • Result: Reflection is not correct
Note
  • The reflections would be correct if:
    • The reflective object is infinitely small
    • The reflected scene is infinitely far away
  • The more you deviate from either of these, the greater the error
  • In practice, bumpy objects hide the inconsistencies
    • Smooth flat objects are the worst
Flaw
  • What if different areas of the scene look very different?
  • Using one environment map for all of them will not look good
  • Solution: Have several environment maps
    • Ex: Half-Life 2 was an early user of this technique
    • Maps were precomputed for several areas around the world
    • Then system used closest one to viewer’s location for reflections
Gloss
  • Rough or glossy objects produce blurred reflections
Gloss
  • We can simulate rough or glossy objects by using a blurred environment map
  • Quick way: Bias the mip level:
    vec3 reflColor = texture(envMap, VR, bias).rgb;
  • Use the roughness lighting parameter to govern how much bias to use
Sky
  • Now that we’ve seen reflective objects, we can turn our attention to the sky
  • Idea: Draw the scene as usual
  • Then: Surround viewer with box (any size is OK)
    • Camera always in center of box
    • Draw box last
    • Use box’s 3D object space coordinates as 3D texture coordinates
    • Sample from cube map texture (sky texture)
Sky
  • Skybox must appear further away than everything else
  • Viewer cannot ever reach the box
  • CPU-side draw code might be like this:
glClear( ... );
globs->prog1->use();
...draw scene as usual...
globs->skyboxprog->use();
globs->skyboxTexture->bind(0);
globs->unitCube->draw();
VS
  • Rotate by rotation used in view matrix, but don’t translate
  • Easy way: Take incoming position, add eyePos to it
    • Ensures viewer can never reach box but they can turn to see any part of it
  • Set z = w
    • Forces it to yon clip plane
    • Send original object space point to FS
VS
layout(location=0) in vec3 position;
out vec3 v_objPos;
void main(){
    vec4 p = vec4(eyePos + position,1.0);
    p = p * viewMatrix * projMatrix;
    p.z = p.w;
    gl_Position = p;
    v_objPos = position;
}
FS
  • FS is very simple
    • No lighting needed
    • Use v_objPos in texture lookup
    • Output resulting color
One More Thing
  • Don’t forget:
    glEnable(GL_TEXTURE_CUBE_MAP_SEAMLESS);
