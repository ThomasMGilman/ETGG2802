Projective Shadows
Motivation
  • Shadows are important visual cue for determining where objects are
  • Ex: Are these two planes going to crash?
Motivation
  • With shadows, we can tell immediately
Projective Shadows
  • One of the simplest shadow techniques
    • Easy to code
    • Easy to understand
    • Shadow caster can be arbitrarily complex
    • No aliasing (blocky shadows)
  • But some drawbacks:
    • Shadow receiver must be planar
    • Only hard shadows (no penumbras)
Procedure
  • Render scene normally
  • Flatten all objects to ground plane
  • Render again, darkening the ground
Plane
  • Recall: A plane is specified with a normal N and a single point p on the plane
    • Alternately, with a single triangle
    • Can get normal via cross product
  • If N=(A,B,C): Plane is set of all points (x,y,z) such that:
    Ax + By + Cz + D = 0
  • D =  − (Apx + Bpy + Cpz)
Ray
  • Given a ray that starts at point r and goes in direction v: What are all points on the ray?
  • Can express with equation: r + t v
    • t=0: At r
    • t=1: At r+v
    • t=2: At r+2v
Equation
  • Given:
    • Plane (we know A,B,C,D)
    • Light source at (Lx,Ly,Lz)
    • Point q in space: Shadow caster
  • Question: Where will shadow from q lie?
Equation
  • Consider ray starting at light and passing through q
  • Points on ray: L + t v
    • Where v = q-L
  • To make math easier: Assume L=(0,0,0)
    • So points on ray = tq
Equation
  • Where does ray strike plane?
    • Where plane equation satisfied
  • Substitute for x,y,z in plane equation:
    Atqx + Btqy + Ctqz + D = 0
  • Solve for the one unknown (t)
    t = ( − D)/(Aqx + Bqy + Cqz)
Equation
  • Now we can find q’ (q, projected to plane):
  • We know q’ = L + tv = t q
  • Substitute t:
    q’ = ( − Dq)/(Aqx + Bqy + Cqz)
  • This gives us three equations (one each for q’s x, y, z coordinates):
    • qx’ = ( − Dqx)/(Aqx + Bqy + Cqz)
    • qy’ = ( − Dqy)/(Aqx + Bqy + Cqz)
    • qz’ = ( − Dqz)/(Aqx + Bqy + Cqz)
  • Goal: Create matrix M so q’ = qM
Matrix
  • This will do the job:
q’ = [ qx qy qz 1 ]⎡⎢⎢⎢⎢⎢⎣  − D 0 0 A         0  − D 0 B         0 0  − D C         0 0 0 0 ⎤⎥⎥⎥⎥⎥⎦
  • q’ = [  − Dqx  − Dqy  − Dqz Aqx + Bqy + Cqz ]
  • After homogeneous divide: We get the result we want
  • Note: This only works if the light is at the origin
Any Light
  • What if light isn’t at origin?
    • Translate everything by -1 * lightPosition
    • Use the matrix given previously
    • Un-translate
Details
  • Let T = translation matrix by -1 * lightPosition
  • Let T-1 be inverse of T
  • Let p be any point on ground (shadow receiver); let N be ground normal
  • Compute p’ = pT
  • Compute A,B,C,D from p’ and N (not original p!)
  • Compute M:
⎡⎢⎢⎢⎢⎢⎣  − D 0 0 A         0  − D 0 B         0 0  − D C         0 0 0 0 ⎤⎥⎥⎥⎥⎥⎦
Matrix
  • Final flattening matrix F =  − (TMT − 1)
    • Why negated?
    • If we multiply a point by F: We get negative w value in gl_Position
    • But GL requires that -w < w
    • By negating whole thing, we ensure GL is happy
    • After homogeneous divide: No difference
Directional
  • What if light is directional? (I.e., infinitely far away)
  • Given: Plane (A,B,C,D)
  • Given: Incoming light direction: (Lx, Ly, Lz)
  • Given: Point q in space
  • Where will shadow from q fall?
  • Consider ray that starts at q and goes in direction L
    • Points on ray = q+tL
Directional
  • Where does ray strike plane?
    • Where plane equation satisfied
  • Substitute: ray points (q+tL) for x,y,z in plane equation
  • Then solve for t:
    • A(qx + tLx) + B(qy + tLy) + C(qz + tLz) + D = 0
    • Aqx + AtLx + Bqy + BtLy + Cqz + CtLz + D = 0
    • t(ALx + BLy + CLz) =  − (Aqx + Bqy + Cqz + D)
    • t = ( − (Aqx + Bqy + Cqz + D))/(ALx + BLy + CLz)
Directional
  • Intersection point = q’ = q + tL
  • Substitute t into formula
    • Let Q = ALx + BLy + CLz to shorten things a bit
    • q’x = qx + ( − (Aqx + Bqy + Cqz + D)Lx)/(Q)
    • q’y = qy + ( − (Aqx + Bqy + Cqz + D)Ly)/(Q)
    • q’z = qz + ( − (Aqx + Bqy + Cqz + D)Lz)/(Q)
Directional
  • This is our matrix F:
( − 1)/(Q)⎡⎢⎢⎢⎢⎢⎣ LxA − Q LyA LzA 0         LxB LyB − Q LzB 0         LxC LyC LzC − Q 0         LxD LyD LzD  − Q ⎤⎥⎥⎥⎥⎥⎦
  • Where Q = ALx + BLy + CLz
Draw
void drawAllObjects(){
    ...
}
void draw(){
    glClear( ... );
    Program::setUniform("flattenMatrix", mat4::identity() );
    Program::setUniform("doingShadow", 0.0 );
    drawAllObjects();
    Program::setUniform("flattenMatrix", F );
    Program::setUniform("doingShadow", 1.0 );
    drawAllObjects();
}

Problems
  • We have two problems:
    • Shadows (shown in red to make more visible) extend outside the shadow receiver
    • Z-fighting
Solution
  • We can use the stencil buffer to fix both problems
  • At setup time: glEnable(GL_STENCIL_TEST);

Draw
  • Set stencil function to always pass and set stencil ops so buffer won’t be changed
  • Draw everything except ground plane
void draw(){
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT | GL_STENCIL_BUFFER_BIT );
    Program::setUniform("flattenMatrix", mat4::identity() );
    Program::setUniform("doingShadow", 0.0 );
    glStencilFunc(GL_ALWAYS,0,0xff);
    glStencilOp(GL_KEEP, GL_KEEP, GL_KEEP);
    drawNonGroundPlaneObjects();
}
Result
  • Color buffer and stencil buffer
Draw
  • Mark area where ground is visible:
    • Set stencil ref to 1: glStencilFunc(GL_ALWAYS,1,0xff);
    • Set stencil ops: glStencilOp(GL_KEEP, GL_KEEP, GL_REPLACE);
  • Draw ground
  • Result: Stencil buffer = 1 where ground exists and is not obscured by other objects

Results
Draw
  • Now we want to draw the shadows
    • Set stencil op to keep/keep/keep
      • This keeps buffer from being changed
    • Set stencil func to EQUAL: glStencilFunc(GL_EQUAL,1,0xff)
      • This draws only where the ground was not obscured by other objects
    • Set flatten matrix and doingShadow flag
    • Disable depth test (stencil buffer will handle overlapping objects)
    • Draw non-ground objects

    drawGroundPlane();
    glStencilOp(GL_KEEP, GL_KEEP, GL_KEEP);
    glStencilFunc(GL_EQUAL,1,0xff);
    Program::setUniform("flattenMatrix", F );
    Program::setUniform("doingShadow", 1.0 );
    glDisable(GL_DEPTH_TEST);
    drawNonGroundPlaneObjects();
    glEnable(GL_DEPTH_TEST);
}
Results
Problem
  • Shadows are usually not completely black
  • Have FS output a value such as (0,0,0,0.5) instead of (0,0,0,1)
  • Blending → Be sure to use GL_SRC_ALPHA / GL_ONE_MINUS_SRC_ALPHA for mode
Problem
  • Artifacts
    • If several shadows overlap: Overdraw → Darker
    • Thin artifact where two faces of cube don’t quite project to same place (look closely!)
  • We have specular highlights in shadows (shouldn’t happen)
Solution
  • Tweak the stencil operation:
    • When drawing shadows, set stencil ops to keep/keep/incr
  • Stencil buffer can now have three values:
    • 0=outside ground plane; cannot receive shadow
    • 1=On ground plane; can receive shadow
    • 2=On ground plane but already received shadow; no more shadows allowed
  • Value can never be larger than 2
Results
Specular
  • To fix specular-highlights-in-shadows:
    • When drawing ground plane: Use diffuse lighting only
    • When done drawing shadows: Add new code
      • Leave stencil op as keep/keep/keep
      • Leave stencil func as glStencilFunc(GL_EQUAL,1,0xff)
      • Set blend func: GL_ONE, GL_ONE
      • Draw ground plane with only specular illumination
    • Result: Only draw specular where no shadows cast
Problem
  • Projection will cast shadows from things that are underground or behind the light source!
Solution
  • Option 1: Don’t let anything be below the ground or behind the light source
    • Behind = “From the ground’s perspective”
    • Easy, and often feasible
  • Option 2: Clip away objects
    • Or: More efficient: Don’t draw objects that are a problem
      • Surround objects with bounding box
      • If box is under ground or behind light source: Discard object
    • If object can go through ground (ex: pool of water), need to clip on per-pixel basis
Clipping
  • New uniform: vec4 groundPlane
    • A,B,C,D of plane equation
  • When doingShadow ≠ 0: In FS:
    • Let p be world space position that we’re considering
    • Compute: dot(groundPlane,vec4(p,1.0))
    • If dot product < 0: Discard fragment (‘discard’ keyword)
  • To inhibit all culling (when not drawing shadows): Set groundPlane to (0,0,0,0)
    • This forces the test to always pass
    • So we don’t need special logic for this case
