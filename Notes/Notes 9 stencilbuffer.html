Stencil Buffer
Purpose
  • Stencil buffer allows us to mask off regions of screen and not draw to them
  • In hardware: Done simultaneously with depth test
  • So it’s “free” time-wise!
Stencil Buffer
  • One entry per pixel
  • Each entry = 8 bit integer
  • We can direct hardware to interpret each integer in a variety of ways
    • Simplest way to use it:
      • zero = don’t draw here
      • nonzero = can draw here
  • But it’s not quite that easy…
Stencil Buffer
  • How do we put our “cutout” in the stencil buffer?
  • Only two ways to change stencil buffer
    • Via a clear: glClear( … | GL_STENCIL_BUFFER_BIT );
    • As a side effect of drawing geometry
Stencil Buffer
  • When GPU is going to draw a fragment, it performs a stencil function on it
  • If function returns true, GL considers stencil test to have passed
  • If function returns false, GL considers stencil test to have failed
Stencil Function
  • Can set stencil function like so:
    glStencilFunc( func, reference, mask )
  • func tells what stencil function to use
  • reference is a reference value (used with function)
  • mask tells which bits of stencil buffer to use
    • Almost always 0xff = 255 = ~0
Stencil Function
  • Stencil function can be:
    • GL_NEVER: Always fails (returns false)
    • GL_ALWAYS: Always succeeds (returns true)
    • GL_EQUAL: Succeeds if reference value equals value in stencil buffer at current pixel location
    • GL_NOTEQUAL: Succeeds if reference ≠ value in stencil buffer
    • GL_GREATER: Succeeds if reference > value in stencil buffer
    • GL_LESS: Succeeds if reference < value in buffer
    • GL_GEQUAL: Succeeds if reference ≥ value in buffer
    • GL_LEQUAL: Succeeds if reference ≤ value in buffer
Visually
Operations
  • We have three operations that we can set: Stencil fail, Depth fail, Pass
    • Stencil fail = Stencil test fails; don’t care what depth test does
    • Depth fail = Stencil test passes but depth test fails
    • Pass = Stencil and depth tests both pass
Operations
  • To set operation: glStencilOp( sfAction, zfAction, zpAction );
    • sfAction = What to do if stencil test fails
    • zfAction = What to do if stencil test passes but z test fails
    • zpAction = What to do if both tests pass
Operations
  • Five choices for each action:
    • Leave stencil buffer alone
    • Invert s.b. value at that pixel location
    • Increment s.b. value
    • Decrement s.b. value
    • Set s.b. value to reference value
Operations
  • Action can be:
    • GL_KEEP: Leave stencil buffer alone
    • GL_ZERO: Set stencil buffer to zero
    • GL_INCR: Increment stencil buffer value; if it overflows, clamp to max value
    • GL_DECR: Decrement stencil buffer value; if it underflows, clamp to 0
    • GL_INVERT: Flip stencil buffer bits (0 leftrightarrow 1)
    • GL_INCR_WRAP: Increment; if it overflows, wrap to zero
    • GL_DECR_WRAP: Decrement; if it underflows, wrap to max value
    • GL_REPLACE: Set stencil buffer to reference value
Setup
  • At setup time, we must enable the stencil test:
    glEnable(GL_STENCIL_TEST);
Uses
  • This may seem pretty abstract.
  • Let’s look at an example use of the stencil buffer
  • Suppose we want to draw a halo around the pumpkin
Preliminaries
  • Add two uniforms:
float forceColorFlag;
vec4 forceColor;
  • If forceColorFlag is nonzero, FS will output forceColor.
  • Otherwise, do shading as usual.
Preliminaries
  • Change FS:
void main(){

        color = forceColor;
        return;
    }
    ...rest is same as usual...
}
Draw
  • Then we need to alter our draw function
  • Suppose M is the worldMatrix for the object we want to halo
void draw(){
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT );
    globs->prog.use();
    globs->camera.setUniforms();
    globs->lightManager.setUniforms();
    Program::setUniform("forceColorFlag",0.0f);
    ...draw all objects...
    Program::setUniform("worldMatrix", scaling(1.05,1.05,1.05) * M );
    Program::setUniform("forceColorFlag",1.0f);
    Program::setUniform("forceColor",vec4(0,1,0,0.25));
    pumpkin->draw();
}
Explanation
  • We draw the scene normally at first
  • Then we draw the haloed object a little bigger on top of itself
Result
  • Not quite what we want
    • One author described the effect as making the object look like it’s “covered in glowing radioactive snot”
    • Yuck!

Step 2
  • Set the stencil test to always pass and don’t let rasterization change the stencil buffer
void draw(){
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT | GL_STENCIL_BUFFER_BIT);
    glStencilFunc(GL_ALWAYS, 1, ~0);
    glStencilOp(GL_KEEP,GL_KEEP,GL_KEEP);
    ...

Problem
  • If we end up rasterizing several overlapping triangles at the last draw, we’ll see some areas with more of the highlight color
    • Because we blend the highlight color repeatedly at some pixels
  • We can fix this easily: Change the end of the draw function:
void draw(){
    ...
    glStencilFunc(GL_EQUAL, 0, ~0);
    glStencilOp(GL_KEEP,GL_KEEP,GL_INCR);       //new!
    Program::setUniform("forceColorFlag",1.0f);
    Program::setUniform("forceColor",vec4(0.4,1,0.4,0.4));
    Program::setUniform("worldMatrix", scaling(1.05,1.05,1.05) * M );
    ...draw haloed object...
}
Explanation
  • Once the halo touches a pixel, the stencil buffer gets set to 1 at that point
  • So it can’t be double-rasterized
Example
  • Draw a diagonal split-screen
Procedure
  • Load two meshes in the shape of the two parts of the split screen
    • I used two triangles
Step 1
  • Modify the draw function
  • First, mask the color and depth buffers
void draw(){
    globs->prog.use();
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT | GL_STENCIL_BUFFER_BIT);
    glColorMask(false,false,false,false);
    glDepthMask(false);
    ...
}
Step 2
  • Next, set the stencil buffer to 1 and draw the first triangle
  • Then set the stencil buffer to 2 and draw the second triangle
void draw(){
    ...
    Program::setUniform("worldMatrix",mat4::identity());
    Program::setUniform("viewMatrix",mat4::identity());
    Program::setUniform("projMatrix",mat4::identity());
    glStencilFunc(GL_ALWAYS, 1, ~0);
    glStencilOp(GL_KEEP,GL_KEEP,GL_REPLACE);
    triangle1->draw();
    glStencilFunc(GL_ALWAYS, 2, ~0);
    triangle2->draw();
    ...
}
Result
  • This is what the stencil buffer looks like (black=0, red=1, green=2)
Step 3
  • Re-enable the color and depth buffers
  • Set the uniforms

  • Draw using camera 1 only where the stencil buffer is 1

Other Examples
  • Example: Effect like Antichamber (TM)
  • Brief outline:
    • Clear screen
    • glStencilOp(GL_KEEP,GL_KEEP,GL_KEEP)
    • glStencilFunc(GL_ALWAYS,0,~0);
    • Draw all cubes. This sets the depth buffer.
    • glStencilOp(GL_KEEP,GL_KEEP,GL_REPLACE)
    • glStencilFunc(GL_ALWAYS,1,~0);
    • Draw one side of one cube
    • glStencilFunc(GL_ALWAYS,2,~0);
    • Draw another side of one cube
    • glStencilFunc(GL_ALWAYS,3,~0);
    • Draw another side of one cube
    • etc.
Other Examples
  • Now SB has each cube face’s visible region mapped out with a different SB value
  • Render each scene: for i = 0 to max:
    • glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    • glStencilOp(GL_KEEP_GL_KEEP,GL_KEEP);
    • glStencilFunc( GL_ALWAYS, i, ~0 );
    • Set camera for scene i
    • Draw meshes for scene i
  • Note that we could use a similar (but simpler) strategy for rendering something like the Portal game.
