Blur
Motivation
  • Many interesting and useful effects are possible if we can render something, blur it, and then do further processing
  • We’ll need to use FBO’s to perform the blurring operation
Terminology
  • Blurring uses a convolution kernel
  • Simplest kernel: Box filter
    • Look an n × n neighborhood of a pixel
    • Average colors together
    • Note: n is always odd
Example
  • Suppose 3x3 kernel
  • For each pixel:
    • Sum up 9 adjacent pixels (a 3x3 square)
    • Multiply result by (1)/(9) to get final result
Blurring
  • What if we want more blurring?
    • Make box bigger
    • Example: 5x5 box
    • Now we sum up 25 adjacent pixels (5x5 square)
    • Multiply end result by (1)/(25)
Problem
  • Number of pixels involved goes up as the square of the box side length
  • This means that time gets prohibitive pretty quickly
  • Most kernels we’re interested in are separable
    • That means we can do horizontal blur followed by vertical blur to get the desired results
  • Much faster: Increases as n, not n2
Box
  • Most basic type of filter: Box filter
    • All samples weighted evenly
  • Intuitively, we’d expect samples near the pixel should get more weight than ones further away
  • We can do this with a Gaussian filter
    • Weights samples more if they’re near the center of the region we’re averaging
Example
  • Gaussian looks better than box, especially for larger kernel sizes
  • 23x23 box vs. 23x23 Gaussian
Gaussian
  • Gaussian formula: If we are i pixels away from the center, the weight of pixel is:
    (eQ)/(σ√(2π))
    • Where Q = (i2)/( − 2σ2)
    • And σ = (r)/(3)
  • Note: i is an integer: 0 ≤ i ≤ radius
Problem
  • Can get artifacts at edge of image if using wrap texture mode
    • Can “fall off” the texture edge and come in on the other side
    • We need a sampler that specifies clamp or mirrored repeat instead
Blurring
  • Since blurring useful in so many contexts, we should add a blur() function to our FBO
  • Strategy: To blur FBO A:
    • Blur A horizontally. Write result to FBO B.
    • Blur FBO B vertically and write result back to FBO A
  • Why two FBO’s?
    • GL doesn’t let us read & write same FBO at once
Blurring
  • Procedure:
    • Make a temporary FBO (FBO B) and set it as render target
    • Set FBO A as the texture source
    • Draw a square over the entire “screen”
    • Then set FBO B as texture source and FBO A as render target
    • Draw square over entire screen
Shader
  • We’ll need a shader to handle blurring
  • VS is very simple:
layout(location=0) in vec3 position;
layout(location=1) in vec2 texCoord;
out vec2 v_texCoord;
void main(){
    gl_Position = vec4(position.xy,-1.0,1.0);
    v_texCoord = texCoord;
}
Shader
  • FS is where the heavy lifting happens
  • Suppose we have some uniforms:
    • vec2 blurDeltas: Will be either (1,0) or (0,1) depending on whether we’re doing horizontal or vertical blur
    • int blurRadius: The size of the blur
    • float blurWeights[MAX_BLUR_SIZE]
      • blurWeights[0] tells weight of pixel under center of convolution kernel
      • blurWeights[i] tells weight of the pixel i units away in either direction
FS
layout(binding=0) uniform sampler2DArray tex;
in vec2 v_texCoord;
out vec4 color;
void main(){
    color = vec4(0);
    color += blurWeights[0] * texture( tex, vec3( v_texCoord , 0 ) );
    for(int i=1;i<=blurRadius;++i){
        float weight = blurWeights[i];
        color += weight * texture( tex, vec3( v_texCoord - i * blurDeltas , 0 ) );
        color += weight * texture( tex, vec3( v_texCoord + i * blurDeltas , 0 ) );
    }
    color.a=1.0;
}

  • The framework’s FBO already provides this code
  • Caches auxiliary blurring FBO’s and Gaussian weights

Radial Blur
  • We can also use blur to get interesting nonrealistic effects
  • For example: Radial blur
  • Procedure:
    • Render scene to FBO
    • Activate radial blur shader
    • Set FBO texture as input
    • Draw quad over whole screen
VS
  • VS is the trivial VS we just saw
FS
  • Useful variables:
    • v_texCoord tells which pixel we’re processing (x,y = 0…1)
    • textureSize(tex) tells width and height of texture ‘tex’
FS
  • Suppose we have this: vec4 c = texture( tex , coord );
  • If we offset coord.x by 1/textureSize.x, we’ll sample one pixel to the right
    • If texture size is NPOT: Might get roundoff error
    • Can use 1.5/textureSize.x to ensure no error
  • If we offset coord.y by 1/textureSize.y, we’ll sample one pixel up
FS
  • To get radial blur:
    • Consider 2D ray that starts at v_texCoord and goes toward (or away from) (0.5,0.5)
      • Length of ray governs amount of blur
    • For several points along the ray (the more points, the smoother the blur)
      • Sample a texel
      • Accumulate into running total
    • Divide runnning total by number of samples
    • Output the color
Depth of Field
  • This seems like a very common effect
  • So we’ll see how to implement it
Procedure
  • Draw scene to FBO 1
  • Copy it to FBO 2
  • Blur one of the FBO’s (I picked FBO 2)
  • Draw quad over whole screen
    • Sample from both FBO textures
    • Blend between blurred and unblurred one
    • That becomes output color
Procedure
  • We already know how to do the blurring
  • What about copying FBO’s?
  • GL has a function for that
glBindFramebuffer( GL_READ_FRAMEBUFFER, globs->fbo.fbo);
glBindFramebuffer( GL_DRAW_FRAMEBUFFER, globs->fbo2.fbo);
glBlitFramebuffer(
    0,0, globs->fbo.width, globs->fbo.height,
    0,0, globs->fbo2.width, globs->fbo2.height,
    GL_COLOR_BUFFER_BIT, GL_NEAREST);
  • Note: We’ll need to change Framebuffer.h to make fbo, width, height public
Note
  • The blit won’t work unless FBO 2 is fully set up
    • Some of the setup happens in the setAsRenderTarget function
  • This is easy to handle: In the main setup() function: Do once:
globs->fbo2.setAsRenderTarget(false);
globs->fbo2.unsetAsRenderTarget();
Draw
  • Our outline for draw():
    • Set globs->fbo as render target
    • Set all uniforms for rendering
    • Draw scene objects as usual
    • Unset globs->fbo as render target
    • Copy globs->fbo to globs->fbo2
    • Blur globs->fbo2
    • Use fboprog
    • glClear
    • Set FBO textures as inputs
      globs->fbo2.texture->bind(0);
      globs->fbo.texture->bind(1);
    • Draw full screen quad
Shader
  • Our FS:
in vec2 v_texCoord;
out vec4 color;

layout(binding=0) uniform sampler2DArray blurred;
layout(binding=1) uniform sampler2DArray unblurred;

void main(){

    vec4 bl = texture( blurred, vec3(v_texCoord,0.0) );
    vec4 unb = texture( unblurred, vec3(v_texCoord,0.0) );

    color = mix( unb, bl, /* something*/ );
    color.a = 1.0;
    return;
}
  • What do we put in for something?
Question
  • What determines amount of blur?
Answer
  • What determines amount of blur?
    • How far we are from focal plane
  • So we need some distance for the focal plane
    • Either a uniform or else hardcode in shader
Question
  • We are in FS
  • Rendering output for some location (x,y)
  • Some object will be visible at that location
  • How do we know how far away the visible object is from the focal plane?
Distance
  • We can pass the FBO’s depth buffer to the shader
  • Make sure to use the unblurred FBO’s depth buffer
  • Example:
    globs->fbo.depthtexture->bind(16);
Question
  • Why 16?
    • As we go through the semester, we’ll use several of the texture units for other purposes
    • All of these will use mipmap filtering
    • But: Depth textures cannot be sampled with linear or mip filtering
      • Must use nearest filtering
      • Otherwise, you get all black
    • Take a look in setup(): We bind a mipmap sampler to texture units 0…15
for(int i=0;i<16;++i)
    globs->sampler.bind(i);
Sampler
  • We need to create a second sampler in Globals:
    Sampler samplerNearest{Sampler::Type::NEAREST, Sampler::Wrap::CLAMP};
  • And then we bind it to texture unit 16 (in setup()):
    globs->samplerNearest.bind(16);
  • Note: 16 is kind of arbitrary
    • I wanted to leave some “headroom” so we don’t need to change other code later
    • You could pick a different number if you want…Just be consistent everywhere
    • Modern GPU’s support at least 96 texture units
    • If you have an older or embedded GPU (cellphone?), you might need to pick a number < 16
Draw
  • Now, inside our draw(), we’ll bind three things, like so:
globs->fbo2.texture->bind(0);
globs->fbo.texture->bind(1);
globs->fbo.depthtexture->bind(16);
Shader
  • We can now code the shader:
in vec2 v_texCoord;
out vec4 color;

layout(binding=0) uniform sampler2DArray blurred;
layout(binding=1) uniform sampler2DArray unblurred;
layout(binding=16) uniform sampler2DArray depth;

void main(){

    vec4 bl = texture( blurred, vec3(v_texCoord,0.0) );
    vec4 unb = texture( unblurred, vec3(v_texCoord,0.0) );
    float d = texture( depth, vec3(v_texCoord,0.0) ).r;

    color = mix( unb, bl, /* something*/ );
    color.a = 1.0;
    return;
}
  • Now what do we put in for “something”?
First Thought
  • Maybe we start with something like this:
float d = texture( depth, vec3(v_texCoord,0.0) ).r;
float focalDistance = 2.0;
float delta = abs(d-focalDistance);
fd = smoothstep( 0.0, 2.5, fd );    //change sharp-focus range
color = mix( unb, bl, fd );
  • This doesn’t work, however
Problem
  • Result from sampling depth texture is a float in range 0…1
  • Where does this value come from?
    • Recall: We had a projection matrix that mapped z to range -1…1
    • GPU then remaps that to range 0…1 for the depth texture
  • We need to undo both mappings to get the true eye-space distance
Computation
  • Recall: Projection matrix results in final z value of -(P+Q/z)
    • Where P = 1 + (2y)/(h − y)
    • And Q = (2hy)/(y)
    • Where h, y are hither and yon
Computation
  • Result of projection matrix -(P+Q/z) gave value in -1…1 range
    • Call it z’
  • GPU then remapped z’: d = (1 + z’)/(2)
    • Notice: If z’=-1, d=0
    • And if z’=1, d=1
So…
  • So: The d value we get back from texture() relates to the original z value thusly:
    d = (1 + z’)/(2) = (1 +  − (P + (Q)/(z)))/(2)
  • We need to solve for z
Solve
  • d = (1 − (P + (Q)/(z)))/(2)
  • 2d = 1 − (P + (Q)/(z))
  • 2d − 1 =  − (P + (Q)/(z))
  • 1 − 2d = P + (Q)/(z)
  • 1 − 2d − P = (Q)/(z)
  • (1)/(1 − 2d − P) = (z)/(Q)
  • (Q)/(1 − 2d − P) = z
Whew!
  • So here’s what we really want in our shader:
float d = texture( depth, vec3(v_texCoord,0.0) ).r;
float z = Q / (1-2*d-P); //RHS: z goes from 0...-infinity
z=-z;
Computation
  • At this point, take absolute difference between z and focal distance
  • Can scale with smoothstep to give faster or slower blurring
  • Then use this in mix()
