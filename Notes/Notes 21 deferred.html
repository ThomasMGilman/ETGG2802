Deferred Rendering
Motivation
  • Consider scene with large amount of overdraw
  • Using our pipeline as we’ve discussed it (forward rendering): Lots of time spent shading pixels that will be overwritten
  • Especially problematic if using costly shading model
Culling
  • Modern GPU’s offer early z culling
    • Before running fragment shader: Determine if pixel is hidden
    • If so: Don’t bother shading
    • This avoids problem of overdraw expense
  • But: Z culling works best if scene drawn front-to-back
    • If scene is in arbitrary order: Doesn’t help as much
Lights
  • Suppose we have several lights in the scene. How to render?
  • Most straightforward way: Shader loops over all lights and adds contributions
  • Idea:
vec3 totalDiffuse=0.0;
for(int i=0;i<MAX_LIGHTS;++i){
    L = lightPositions[i].xyz - v_worldPos ;
    L = normalize(L);
    totalDiffuse += max(0.0, dot(N,L) );
    ...likewise for specular...
}
color = totalDiffuse * textureColor;
Problem
  • What if we want many lights? Suppose a city scene with streetlights.
  • Must loop over all the lights for every fragment
  • Even if the light can’t affect a pixel (attenuation/spotlight)
Deferred Rendering
  • Basic idea: Defer as much work as possible to as late as possible
  • Make every shading computation count: Don’t shade locations that aren’t visible
  • Deferred rendering breaks up scene render into two phases
    • First, for each pixel, figure out which object is visible at that location
    • Second, compute the color for that pixel
Phase 1
  • To determine visibility, we draw scene into a Gbuffer
    • Gbuffer = FBO with several output layers
  • We use the usual VS (and GS, if applicable)
  • But the FS doesn’t output colors
    • Instead, it outputs data
Phase 2
  • Use the Gbuffer as the input texture
  • Draw a quad over the whole screen
  • Use identity VS: Just passes data through
  • FS performs shading computations
    • Sample data from FBO
    • Do lighting calculations
    • Output final color
Analysis
  • Several key benefits
    • Geometry processing totally decoupled from shading computations
    • We never do any needless shading work: Everything that we shade is guaranteed to be visible
    • We can support large number of light sources

Data
  • What data does first pass need to output?
  • Depends on what we need for color computations in second pass
Data
  • Suppose we’re using Lambert for diffuse and Phong for specular
  • We need to know:
    • Position of surface point (world space)
    • Normal at surface point (world space)
    • Shininess exponent
    • Base (diffuse) texture color
    • Specular texture color
    • Emissive texture color
  • For our first example, we’ll ignore bump maps and more complex shading models
  • We also omit any postprocessing (glow, etc.)
Option 1
  • We could create an FBO with 6 RGBA32F attachments…but that’s a bad idea
    • Limitation: How many targets we can have
      • Most desktops allow 8
      • Some mobiles might only allow 4
    • Limitation: Memory
      • 1366 × 768 screen: 16MB per RGBA32F buffer
      • If six buffers: 96MB just for the Gbuffer (96 bytes per pixel)
    • Limitation: Memory bandwidth
      • 60fps: GPU needs to push 5.7GB of data around per second
Option 2
  • Most programs try to pack data and limit overall size
  • We can quantize some data to 16-bit floats or even 8 bit bytes
Layout
  • Position: 32-bit float × 3
  • Normal: 16-bit half × 3
  • Shininess exponent: Either half or else quantized 8 bit
  • Base color: 8-bit rgba
  • Specular color: 8-bit rgb
  • Emissive color: 8-bit rgb
Note
  • 3-channel formats are inconvenient for GPU to work with
  • So some GPU’s only support 1, 2, or 4 channel formats
  • Packing can get pretty crazy sometimes!
Example
  • We might do this:
    • RGBA32F: r,g,b = Position, a=flag (explained later)
    • RGBA16F: r,g,b = Normal, a=shininess
    • RGBA8: r,g,b,a = base color rgba
    • RGBA8: r,g,b = specular rgb, a=emissive r
    • RG8: r,g = emissive g,b
  • Total bytes per pixel: 34

Phase 1

void deferredPhase1(){
    glBlendFunc(GL_ONE,GL_ZERO);
    globs->camera.setUniforms();
    globs->gbuffer.setAsRenderTarget(true);
    drawSceneObjects();
    globs->gbuffer.unsetAsRenderTarget();
}

Shaders

Shaders
  • Here’s what the FS might look like:
in vec2 v_texcoord;
in vec3 v_worldPos;
in vec3 v_normal;

layout(location=0) out vec4 slot0;  //rgb=position, a=flag
layout(location=1) out vec4 slot1;  //rgb=normal, a=shiny
layout(location=2) out vec4 slot2;  //rgba=base color
layout(location=3) out vec4 slot3;  //rgb=specular, a=emissive
layout(location=4) out vec4 slot4;  //rg=emissive

layout(binding=0) uniform sampler2DArray baseTexture;
layout(binding=1) uniform sampler2DArray emissionTexture;
//suppose rgb = specular color, a=shininess
layout(binding=2) uniform sampler2DArray specularTexture;

void main(){
    vec4 baseColor = texture( baseTexture, vec3( v_texcoord, 0.0 ) );
    vec4 emissionColor = texture( emissionColor, vec3( v_texcoord, 0.0 ) );
    vec4 specularColor = texture( specularTexture, vec3( v_texcoord, 0.0 ) );
    slot0 = vec4( v_worldPos, 1.0 );    //flag is 1.0
    slot1 = vec4( v_normal, specularColor.a );
    slot2 = baseColor;
    slot3 = vec4( emissionColor.rgb, specularColor.r);
    slot4 = vec4( specularColor.gb, 0.0, 0.0 );
}

Pass 2
  • Now we set the Gbuffer as the input texture for our second pass shader
  • We render a quad over the whole screen
  • The VS does nothing fancy: Just pass position and texture coordinate through
  • In the FS: We sample from each of the render slots
  • We now know the world space position, normal, colors, and shininess at each pixel
  • Then do the usual lighting computations
  • And output the final color
Pass 2
  • Here’s what the CPU side code might look like for phase 2:
void deferredPhase2(){
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    globs->gbuffer.textures[0]->bind(0);
    globs->gbuffer.textures[1]->bind(1);
    globs->gbuffer.textures[2]->bind(2);
    globs->gbuffer.textures[3]->bind(3);
    globs->gbuffer.textures[4]->bind(4);
    globs->deferredPass2Prog.use();
    globs->fullScreenQuad.draw();
}
Pass 2
  • Pass 2 fragment shader:
in vec2 v_texcoord;
out vec4 color;

layout(binding=0) uniform sampler2DArray slot0Texture;
layout(binding=1) uniform sampler2DArray slot1Texture;
layout(binding=2) uniform sampler2DArray slot2Texture;
layout(binding=3) uniform sampler2DArray slot3Texture;
layout(binding=4) uniform sampler2DArray slot4Texture;

void main(){
    vec4 slot0 = texture( slot0Texture, vec3(v_texcoord,0.0));
    vec4 slot1 = texture( slot1Texture, vec3(v_texcoord,0.0));
    vec4 slot2 = texture( slot2Texture, vec3(v_texcoord,0.0));
    vec4 slot3 = texture( slot3Texture, vec3(v_texcoord,0.0));
    vec4 slot4 = texture( slot4Texture, vec3(v_texcoord,0.0));
    vec3 v_worldPos = slot0.xyz;
    vec3 N = slot1.xyz;
    float shiny = slot1.w;
    vec4 diffuseColor = slot2;
    vec3 emitColor = slot3.rgb;
    vec3 specularColor = vec3(slot3.a, slot4.rg );
    vec3 L = normalize( lightPosition.xyz - lightPosition.w * v_worldPos);
    vec3 V = normalize( eyePos - v_worldPos );
    vec3 R = reflect( -L, N );
    float diffuse = lambert( L,N, diffuseColor.rgb );
    float specular = phong( L,N,V, shininess );
    color.rgb = diffuse + specular + emitColor;
    color.a = 1.0;
}

Question
  • What about the FS’s for things that aren’t illuminated?
    • Ex: Skybox, stars, billboards…
Solution
  • For skybox, write 0.0 to flag (slot0.a)
  • Then write skybox color to slot 2 (base color)
  • Rest of slots are ignored; just write zeros to them
  • In second pass FS, look at slot0Texture alpha
    • If 1.0: Do computations shown previously
    • If 0.0: Copy slot2 color to output and return
Bump Mapping
  • If we’re using bump mapping, the bump mapped normal needs to be computed in the first-pass fragment shader
  • This goes against our “defer as much work as possible until the last minute” philosophy
  • But: Computing the mapped normal requires knowing the world matrix and the tangent vector
    • That would mean 9 additional floats to write (only need 3x3 for vector transform)
    • So we do the bump-mapping work up front
  • Sometimes, we have to compromise our approach to deal with hardware limitations
Bump Mapping
  • Another approach would be to use a matrix palette
    • Create a “texture” with data for every world matrix used when drawing the frame
      • Ex: Suppose we had 100 different world matrices.
      • Texture would have width 4 and height 100
      • Each “row” of texture has 16 floats (rgbα); that’s enough for a 4x4 matrix
        • We really only need the upper 3x3 (rotation) part
    • In phase 1: Output a single float to a Gbuffer slot corresponding to the row number of the relevant matrix
    • In phase 2: Read the matrix row and pull the data so we can do bump mapping
  • We won’t use this approach here since it’s a bit more complex
Multiple Lights
  • What if we have multiple lights?
  • We break the second phase into two parts: Phase 2A and 2B
    • In phase 2A, we draw a quad over the whole screen and render emissive colors only
    • In phase 2B, we do a loop where we accumulate the contribution from each light source
Render
  • Overall draw function looks like this:
void draw(){
    globs->camera.setUniforms();
    globs->lightManager.setUniforms();  //only used in phase 2B...
    deferredPhase1();
    deferredPhase2A();
    deferredPhase2B();
}

  • Neither CPU code nor shaders change for pass 1
  • So we don’t need to look at these again
Phase 2
  • There are changes to both CPU and GPU side code
  • First, we’ll look at the CPU code

Phase 2B
  • And now the CPU code for phase 2B
    • Change the uniform
    • Draw a quad for each light source
    • This will sucessively add each light source’s contribution on top of whatever’s already there
void deferredPhase2B(){
    for(int i=0;i<numLights;++i){
        Program::current->setUniform("renderPhase",(float)i);
        globs->fullscreenQuad.draw();
    }
}
FS
  • Second pass VS doesn’t change
  • Only the FS is different
in vec2 v_texcoord;
out vec4 color;

layout(binding=0) uniform sampler2DArray slot0Texture;
layout(binding=1) uniform sampler2DArray slot1Texture;
layout(binding=2) uniform sampler2DArray slot2Texture;
layout(binding=3) uniform sampler2DArray slot3Texture;
layout(binding=4) uniform sampler2DArray slot4Texture;

void main(){
    vec4 slot0 = texture( slot0Texture, vec3(v_texcoord,0.0));
    vec4 slot1 = texture( slot1Texture, vec3(v_texcoord,0.0));
    vec4 slot2 = texture( slot2Texture, vec3(v_texcoord,0.0));
    vec4 slot3 = texture( slot3Texture, vec3(v_texcoord,0.0));
    vec4 slot4 = texture( slot4Texture, vec3(v_texcoord,0.0));
    if( renderPhase == -1.0 ){
        //pass 2A: Only unshaded objects
        color = slot2;
        return;
    }
    //if we get here, we're in pass 2B: Only lit objects
    if( all( equal( vec3(0.0),slot1.xyz) ) ){
        //it's an unlit object. Don't render again.
        color = vec4(0);
        return;
    }
    vec4 lightPos = lightPositions[int(renderPhase)];       //new!
    ...do calculations as usual except don't add emissive...
}
Speedup
  • We can speed up the lighting computations considerably
  • In phase 2B: Instead of drawing a quad over the whole screen:
    • Figure out light’s bounding box (the region the light can influence)
    • Often, lights are stationary and have limited range of influence (
      • Attenuation or spotlights
  • Enable face culling: glEnable(GL_CULL_FACE)
  • Draw the light’s bounding volume (point light → cube; spotlight → cone or pyramid)
  • The only pixels that we touch are those ones that the light could affect
  • Speeds up render time
Postprocessing
  • Where do postprocessing effects (glow, blur, etc.) fit in?
  • We do them after we’ve done the deferred second phase render
  • Suppose we want to apply glow effect
void draw(){
    deferredPhase1();
    glClear( GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    deferredPhase2A();
    deferredPhase2B();
    globs->gbuffer.blur( 3, 11.0f, 2.0f );  //layer, radius, multiplier
    glBlendFunc(GL_ONE,GL_ONE);
    ...draw quad textured with layer 3 of gbuffer...
    ...draw HUD, etc....
}
Billboards
  • How do billboards fit in?
  • That’s a little trickier…
  • α transparency is difficult with deferred rendering
  • Why?
Transparency
  • Problem: When we do transparency, we normally compute color of incoming fragment (source) and then blend (mix) with color of fragment already on the screen (destination)
  • But in deferred rendering, we don’t know either source or destination fragment colors in pass 1
  • And in pass 2, we only have information about the fragment that’s closest to the viewer, so there’s nothing to blend it with!
Transparency
  • We can incorporate transparent objects, but we have to render them using another pass
    • Use forward rendering
  • So we have three render phases
    • Deferred phase 1
    • Deferred phase 2A and 2B
    • Forward render phase
Procedure
  • During deferred phase 2A, we’ll pull data from the Gbuffer’s depth texture and write it to the destination’s depth buffer
  • Depth textures can only be accessed in GL_NEAREST texture mode (mipmapping is not allowed)
  • We must create a Sampler to use GL_NEAREST mode
class Globals{
    Sampler mipSampler;    //mipmapping, nearest
    Sampler nearestSampler{Sampler::Type::NEAREST};
}
Setup
  • Set the sampler
globs->nearestSampler.bind(5);
Draw
  • Phase 1: No changes (except: Only draw opaque objects)
  • Store data (color, normal, position, etc.) to GBuffer
Draw
  • Phase 2A: We need to use the non-mipmapped sampler and bind the depth texture as a shader input
void deferredPhase2A(){
    globs->gbuffer.depthtexture->bind(1);
    glBlendFunc( GL_ONE, GL_ONE );
    //clear color MUST be black!
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    globs->gbuffer.textures[0]->bind(0);
    //...bind textures[1,2,3] too...
    globs->gbuffer.textures[4]->bind(4);
    globs->gbuffer.depthtexture->bind(5);
    globs->deferredPass2Prog.use();
    //-1 = draw emissive only
    Program::current->setUniform("renderPhase",-1.0f);
    globs->fullscreenQuad.draw();
}
Shader
  • The phase 2 fragment shader will need a new global:
    layout(binding=5) uniform sampler2DArray depth_texture;
  • In FS main():
if( renderPhase == -1.0 ){
    //drawing emissive only
    color = slot3;
    gl_FragDepth = texture( depth_texture, vec3(v_texcoord,0) ).r;
    return;
} else {
    gl_FragDepth = 0;
}
Phase 2B
  • Phase 2B: We mask the depth buffer so it doesn’t get changed
void deferredPhase2B(){
    glDisable(GL_DEPTH_TEST);
    for(unsigned i=0;i<lightPositions.size();++i){
        Program::current->setUniform("renderPhase",(float)i);
        globs->usq.draw();
    }
    glEnable(GL_DEPTH_TEST);
}
Phase 3
  • Finally, we add code to do phase 3
  • Use ordinary forward rendering here
void deferredPhase3(){

}
