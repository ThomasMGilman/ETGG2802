Planar Reflections
Motivation
  • We’ve seen the use of environment maps to simulate reflective objects
  • But these are not good for reflective planar objects
    • Geometric inaccuracies
  • Here, we’ll look at ways to simulate planar reflections
Example
  • Here’s our example scene
  • We want the grey square to be reflective
Procedure
  • Draw the scene as usual
  • Reflect all the objects around the plane of the mirror
  • Then draw the scene again
  • Finally, draw the mirror (a quad or similar)
Question
  • How do we reflect objects around the mirror’s plane?
  • Given an arbitrary plane: Want to reflect p around it to get p’
Math
  • Recall: Any plane can be described by four constants A,B,C,D
    • (A,B,C) = normal to plane
    • D essentially expresses distance from plane to origin
  • Equation: f(p) = Apx + Bpy + Cpz + D
    • f(p) tells signed distance from p to plane
      • If p is on the plane, f(p)=0
      • If p is in front of plane, f(p)>0
      • If p behind plane, f(p)<0
Plane Equation
  • Suppose we know the normal to the plane
  • Suppose we know one point m on the plane
  • Then Amx + Bmy + Cmz + D = 0
  • So D =  − (Amx + Bmy + Cmz)
Plane Equation
  • If all we know is a single triangle v0v1v2 that lies within the plane:
    • Plane normal = triangle normal
    • Compute: N = (v1 − v0) × (v2 − v0)
Reflection
  • Given point p + the plane normal N + plane D value: Compute reflected point p’
    • Let d = N ⋅ p + D
      • Ax+By+Cz+D for p → Gives distance of p from plane
    • p’ = p + -d N + -d N = p + -2 d N
Reflection
  • p’ = p + -2 d N
  • Expand d out:
    • p’ = p + -2 (N ⋅ p + D) N
  • Distribute the N:
    • p’ = p + -2 ( (N ⋅ p) N + D N)
  • Distribute the 2:
    • p’ = p + -2 (N ⋅ p) N + -2 D N
Reflection
  • Goal: We’d like to make a matrix R such that p’ = p R
  • How can we do this?
Reflection
  • We know R must be a 4x4 matrix
  • So that’s a start…
p’ = pR = p ⋅ ⋅ ⋅ ⋅         ⋅ ⋅ ⋅ ⋅         ⋅ ⋅ ⋅ ⋅         ⋅ ⋅ ⋅ ⋅
Reflection
  • We had:
    • p’ = p +  − 2(N⋅p)N +  − 2DN
  • But p and N are vec3’s. So we really have three equations:
    • px’ = px +  − 2(N⋅p)Nx +  − 2DNx
    • py’ = py +  − 2(N⋅p)Ny +  − 2DNy
    • pz’ = pz +  − 2(N⋅p)Nz +  − 2DNz
Matrix
  • If we have p’ = p R: We know that p’x comes from first column of R
  • px’ = px +  − 2(N⋅p)Nx +  − 2DNx
  • There’s a constant term. Only one place for it.
p’ = pR = p⎡⎢⎢⎢⎢⎢⎣ ⋅ ⋅ ⋅ ⋅         ⋅ ⋅ ⋅ ⋅         ⋅ ⋅ ⋅ ⋅          − 2DNx ⋅ ⋅ ⋅ ⎤⎥⎥⎥⎥⎥⎦
Matrix
  • We have px’ = px +  − 2(N⋅p)Nx +  − 2DNx
  • Expand the dot product out:
    • px’ = px +  − 2(pxNx + pyNy + pzNz)Nx +  − 2DNx
  • Distribute the  − 2Nx
    • px’ = px +  − 2NxpxNx +  − 2NxpyNy +  − 2NxpzNz +  − 2DNx
  • Rearrange terms:
    • px’ = px + px( − 2N2x) + py( − 2NxNy) + pz( − 2NxNz) +  − 2DNx
  • Factor px from the first two terms:
    • px’ = px(1 − 2N2x) + py( − 2NxNy) + pz( − 2NxNz) +  − 2DNx
  • We can now get the matrix’s first column! (Do you see how?)
Matrix
R = ⎡⎢⎢⎢⎢⎢⎣ 1 − 2N2x ⋅ ⋅ ⋅          − 2NxNy ⋅ ⋅ ⋅          − 2NxNz ⋅ ⋅ ⋅          − 2DNx ⋅ ⋅ ⋅ ⎤⎥⎥⎥⎥⎥⎦
Matrix
  • Second and third columns found similarly
  • w must be 1, so last column is 0,0,0,1
Matrix
  • End result (you can verify it’s correct, if you like):
R = ⎡⎢⎢⎢⎢⎢⎣ 1 − 2N2x  − 2NxNy  − 2NxNz 0          − 2NxNy 1 − 2N2y  − 2NyNz 0          − 2NxNz  − 2NyNz 1 − 2N2z 0          − 2DNx  − 2DNy  − 2DNz 1 ⎤⎥⎥⎥⎥⎥⎦
Matrix
  • Or, a more concise way of denoting it: If I = identity matrix:
R = I − 2⎡⎢⎢⎢⎢⎢⎣ N2x NxNy NxNz 0         NxNy N2y NyNz 0         NxNz NyNz N2z 0         DNx DNy DNz 0 ⎤⎥⎥⎥⎥⎥⎦
Code
  • How can we do this in code?
  • Add a new uniform: mat4 reflectionMatrix
  • Then tweak the vertex shader:
void main(){
    vec4 p = vec4(position,1.0);
    p = p * worldMatrix;
    v_worldPos = p;
    p = p * reflectionMatrix;
    p = p * viewMatrix;
    p = p * projMatrix;
    ...
}
  • Need to use unreflected position for lighting computations (v_worldPos)

Result
  • Oops. What’s wrong?
Problem
  • First problem: Suppose we have an object that’s behind the mirror
    • When we reflect it, it ends up in front of mirror
    • And it gets rendered there
  • In second draw pass, must cull away these objects
  • If we have bounding volumes, can do at CPU level
  • If objects can pass through mirror, will also need to do at shader level
Culling
  • Suppose object has a bounding sphere with center c and radius r
  • Let reflection plane be defined by A,B,C,D
  • If Acx + Bcy + Ccz + D <  − r then object was behind reflection plane
  • So don’t draw it
Culling
  • What if we need to cull at shader level?
  • This is handled in FS
  • Pass reflection plane as a uniform: vec4 reflPlane
  • Let p be world space position that we’re considering
    • Compute: dot(reflPlane,vec4(p,1.0))
    • If dot product < 0: Discard fragment (‘discard’ keyword)
  • To inhibit all culling (as on first draw call): Set reflPlane to (0,0,0,0)
    • This forces the test to always pass
Results
  • We’ve fixed one problem
Fix #2
  • When we draw the reflective object (the mirror), it obscures the reflection
  • We need to set α < 1 when we draw mirror
Problems
  • Two problems left:
    • Things that are behind the mirror are visible “through” it
      • Ex: The cylinder
    • Reflections can “poke out” from behind the mirror
      • Ex: The cone
  • We’ll solve these next time!
