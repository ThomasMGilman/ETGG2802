Physically Based Rendering
Motivation
  • Want more realistic lighting
    • Remember: Phong lighting was not energy conserving
  • Want more physically based rendering (PBR) model
Example
  • Motivating demo: link
Definitions
  • Before we begin: We’ll need some definitions
  • Some vectors: All are unit length:
    • V=vector from surface to viewer
    • L=vector from surface to light source
    • N=normal
    • H=L+V (the “half vector”)
Definitions
  • We also have some angles:
    • θo = Angle between N and V (“outgoing”)
    • θi = Angle between N and L (“incoming”)
    • φ = Angle between V and H
    • ψ = Angle between H and N
Math
  • Several of the computations we’ll look at require cos θo,
    cos θi, sin θo, and sin θi
  • We can get these with dot products and using the identity sin2θ + cos2θ = 1 (for any θ):
    • cos(θo) = N⋅V
    • cos(θi) = N⋅L
    • sin(θo) = √(1 − (cosθo)2)
    • sin(θi) = √(1 − (cosθi)2)
    • cos(φ) = V⋅H
    • cos(ψ) = N⋅H
Review
  • First, we’ll review our basic lighting model from 2801
  • We considered two components: Diffuse and specular
    • Diffuse = general reflection; no preferred direction
    • Specular = mirror-like reflection; preferentially oriented
Lambert
  • Physically correct for perfect diffuse reflectors
    • Lambert’s cosine law
    • Although…Most surfaces aren’t perfect diffuse reflectors
    • Lambert’s law is an approximation
  • We could write this in code like so:
vec3 lambert(float cos_theta_i, vec3 baseColor){
    return max(0.0,cos_theta_i) * baseColor;
}
Phong
  • Recall the Phong model:
    • Compute the reflection vector R of L about N
    • Compute the cosine of the angle between R and V
    • Raise that cosine to a power
  • In code:
vec3 phong(float cos_theta_i, vec3 L, vec3 N, vec3 V,
           float shininess, vec3 baseColor){
    if( cos_theta_i <= 0.0 )
        return vec3(0.0);
    vec3 R = reflect(-L,N);
    float dp = max(0.0,dot(V,R));
    //optional: Don't multiply by baseColor
    return baseColor * pow( dp, shininess );
}
Combination
  • How do we combine diffuse and specular?
  • And what if we have several light sources?

Lighting
  • If we have several lights, we need to compute the illumination for each one separately
  • We could do it something like this:
void computeIllumination( vec3 lightPosition, float posOrDir,
                          vec3 spotDirection, float cosMaxSpotAngle,
                          float cosSpotFadeAngle, vec3 lightColor,
                          vec3 N, vec3 V, float shininess,
                          vec3 baseColor,  out vec3 diffuseAmount,
                          out vec3 specularAmount )
{
    vec3 L = lightPosition.xyz - posOrDir * v_worldPos;
    //prevent division by zero
    float lightDistance = max(0.0001,length(L));
    L = 1.0/lightDistance * L;
    float attenuation = 1.0 / (A[0] + lightDistance * (A[1] + lightDistance * A[2]));
    float cos_theta_i = dot(N,L);
    float spotDot = dot(-L, spotDirection );
    float spotF = clamp( (spotDot-cosineMaxSpotAngle) /
        (cosineSpotFadeAngle-cosineMaxSpotAngle), 0.0, 1.0 );
    vec3 tmp = spotF * lightColor * attenuation;
    diffuseAmount = tmp * lambert( cos_theta_i, baseColor );
    specularAmount = tmp * phong( cos_theta_i, L, N, V, shininess, baseColor );
}
Lighting
  • Computing the total contribution from all the lights would look like so:
void computeTotalIllumination( float shininess, vec3 baseColor,
    out vec3 totalDiffuseAmount, out vec3 totalSpecularAmount)
{
    totalDiffuseAmount = vec3(0);
    totalSpecularAmount = vec3(0);
    vec3 N = normalize(v_normal);
    vec3 V = normalize( eyePos - v_worldPos );
    for(int i=0;i<MAX_LIGHTS;++i){
        vec3 diff, spec;
        computeIllumination( lightPositions[i].xyz,
            lightPositions[i].w, spotlightDirections[i].xyz,
            spotlightDirections[i].w,  lightColors[i].w,
            lightColors[i].xyz, N, V, shininess,
            baseColor, diff, spec );
        totalDiffuseAmount += diff;
        totalSpecularAmount += spec;
    }
}
Lighting
  • Finally, we’d use the function like so:
void main(){
    vec3 totalDiffuseAmount = vec3(0);
    vec3 totalSpecularAmount = vec3(0);
    float shininess = ...some value... ;
    vec4 baseColor = texture( baseTexture, vec3(v_texCoord,0.0) );
    computeTotalIllumination( shininess, baseColor.rgb, totalDiffuseAmount,
        totalSpecularAmount );
    vec4 emitColor = texture( emissionTexture, vec3(v_texCoord,0.0) );
    color.rgb = ambient * baseColor.rgb +
          totalDiffuseAmount + totalSpecularAmount +
          emitColor.rgb;
    color.a = baseColor.a;
}
Fresnel
  • Consider these two geometries:
  • Phong’s model says specular illumination depends only on angle between V and R
  • So we’d get same specular illumination in both cases
Real Life
  • That’s not how objects work in real life!
  • Imagine a road (wet asphalt) or the ocean’s surface
    • You’re driving at sunset, into the sun
      • How reflective is the road?
    • It’s high noon, and you’re looking down at the road
      • How reflective is it?
  • This phenomenon is called Fresnel effect
    • Pronounced “fray-nell” or “frah-nell”
    • Phong model doesn’t account for it
    • But it’s important for realistic surface rendering
Roughness
  • No surface is truly smooth
  • Small-scale surface roughness: Microfacets
    • Responsible for subtle but important lighting effects
    • Microfacets interact with light in several ways…
Facets
  • Some facets will shadow incoming light
Facets
  • Some facets will block reflected light
Facets
  • Some facets interreflect light multiple times, affecting color on each bounce
Metals
  • Metals have (nearly) zero diffuse reflectivity
    • Virtually 100% of their appearance is due to specular reflections
  • We must also account for this if we want realism
Luna
  • Frank Luna presents a model that implements a simple approximation to the microfacet scheme
  • Note that this only addresses specular – not diffuse
  • Here’s an adaptation of his equation: F⋅ (S + 8)/(8)⋅(cosψ)S
    • F = Fresnel value
    • S = Shininess value
      • Kind of ad hoc
Fresnel
  • We can compute Fresnel amount exactly (based on physics), but the math is costly
  • Usually, we use an approximation; several exist
  • Schlick introduced one widely used approximation: F = ((1 − cosφ)5)(1 − F0) + F0
    • Where F0 = Amount of reflectance at 0° incident angle
F0
  • What is the value of F0?
    • Depends on your material
  • F0 is usually treated as a vec3 (r,g,b)
  • Ex: For gold: A yellowish color: Very high red, high green, low blue
  • For nonmetals: It’s usually small greyscale value (usually less than 0.25)
Examples
  • Some examples: From Hoffman’s presentation (see sources)
    • Chromium: 0.77,0.0.77,0.77
    • Copper: 0.98, 0.82, 0.76
    • Gold: 1.0, 0.90, 0.62 or 1.02, 0.78, 0.34
    • Silver: 0.99, 0.98, 0.96
    • Aluminum: 0.96, 0.96, 0.96
    • Water: 0.15, 0.15, 0.15
    • Plastic or glass: 0.23, 0.23, 0.23
    • Crystal: 0.27, 0.27, 0.27
    • Diamond: 0.43, 0.43, 0.43
Code
  • We could code this like so:
vec3 lunaFresnel(float cos_phi, vec3 F0){
    vec3 one_minus_F0 = vec3(1.0)-F0;
    return F0 + one_minus_F0 * pow(1.0 - cos_phi,5.0);
}
vec3 luna(float cos_psi, float cos_phi, vec3 baseColor, float shininess){
    vec3 F = lunaFresnel(cos_phi, baseColor  );
    return F * (shininess+8.0)/8.0 * pow(cos_psi,shininess);
}
  • In computeIllumination, replace the call to phong() with luna()
Analysis
  • This is more realistic looking than Phong shading for metallic objects
  • One drawback: This scheme isn’t totally physically based
    • The shininess value is more-or-less arbitrary
    • But at least we are factoring in Fresnel

Schlick
  • Schlick produced a more complex model for PBR
    • Others have tweaked the model over time
  • We have two parts: diffuse and specular
Specular
  • specular = cosθi ⋅ F ⋅ V ⋅ D
    • F = Fresnel factor (vec3: r,g,b)
    • V = Visibility factor (self shadowing of microfacets; scalar)
    • D = Microfacet distribution factor; scalar
F
  • The Fresnel factor in this model is a bit more complex than that given previously
  • Let F0 = mix( vec3(0.04), objectColor, μ )
    • Where μ is metallicity constant
    • 0 = totally not metallic; 1=totally metallic
    • Idea: Nonmetallic objects have very small Fresnel reflectance (but never quite zero)
    • Metallic objects have more Fresnel reflectance
  • Then compute F as before
    F = ((1 − cosφ)5)(1 − F0) + F0
V
  • The Smith Joint GGX approximation:
V = (1)/(2(cosθi√(ρ2 + (1 − ρ2)cos2θo) + cosθo√(ρ2 + (1 − ρ2)(cos2θi)))
  • Where ρ is a roughness constant (0…1)
    • ρ = 0 → Very smooth at microfacet level
    • ρ = 1 → Very rough at microfacet level
D
  • The Trowbridge-Reitz formula:
D = (1)/(π)⎛⎝(ρ)/(1 + (cos2ψ)(ρ2 − 1))⎞⎠2
Diffuse
  • We need to scale the diffuse amount so the model is energy conserving
  • d = mix( 0.96*baseColor , vec3(0), μ )
    • μ is the metallicity constant (0…1) discussed previously
    • Idea: The more metallic the object is, the less diffuse reflectance
    • baseColor is a vec3 (r,g,b)
  • diffuse = cos(θi)⋅(1 − F)⋅(d)/(π)
    • F is the Fresnel factor (a vec3: r,g,b)
    • diffuse is also a vec3 (r,g,b)
    • The division by π is “physically correct”
    • Some authors omit the π divisor and scale the baseColor input instead
Code
  • We can compute the Fresnel factor like so:
vec3 schlickFresnel(float cos_phi, vec3 objectColor, float metallicity){
    vec3 F0 = mix( vec3(0.04), objectColor, metallicity );
    vec3 one_minus_F0 = vec3(1.0)-F0;
    return F0 + one_minus_F0 * pow(1.0 - cos_phi,5.0);
}
Code
  • Then we have the diffuse computation
    • Replace the call to lambert() with schlickDiffuse()
//F=schlickFresnel(); mu=metallicity
vec3 schlickDiffuse( vec3 F, float cos_phi, float mu, vec3 baseColor,
                     float cos_theta_i )
{
    vec3 d = mix( 0.96*baseColor , vec3(0), mu );
    //might want to divide by pi here
    return cos_theta_i * ( vec3(1.0)-F) * d ;
}
Code
  • Finally, the specular
    • Replace the call to phong() with schlickSpecular()
//F=schlickFresnel(); rho = roughness; mu=metallicity
vec3 schlickSpecular( vec3 F, float cos_phi, float cos_psi,
        vec3 baseColor, float cos_theta_i,
        float cos_theta_o, float rho, float mu)
{
    float rho2 = rho*rho;
    float V = 1.0 / (
        2.0 * (
            cos_theta_i * sqrt( rho2 + (1.0-rho2) * cos_theta_o * cos_theta_o ) +
            cos_theta_o * sqrt( rho2 + (1.0-rho2) * cos_theta_i * cos_theta_i ) ) );
    float tmp = rho / (1.0 + cos_psi*cos_psi * (rho2-1.0) );
    float D = 1.0/PI * tmp*tmp;
    return cos_theta_i * F * V * D;
}

Other Models
  • Schlick’s model is widely used today for specular reflectance
  • But it uses Lambert’s model for the diffuse component
    • Lambert’s cosine law is only true for perfect diffuse reflectors
    • Many surfaces are not truly Lambertian
  • Oren & Nayar developed an alternate model
    • Assumes that surface has microfacets that can reflect light back to viewer (retroreflection)
Oren-Nayar
  • diffuse = (cosθi)(A + Bcos(α)sin(a)tan(b))
  • We’ll unpack these parameters in the following slides…
Parameters
  • A = 1 − (1)/(2)⋅(σ2)/(σ2 + 0.33)
  • B = (9)/(20)⋅(σ2)/(σ2 + 0.09)
    • σ is a roughness parameter (user-supplied; specific to the surface material)
      • σ = 0 → Same as Lambertian surface
      • σ = 1 → Very rough surface: More retroreflection
Parameters
  • Suppose we project L (vector to light) to the surface and we also project V (vector to viewer) to the surface.
  • Let α be the angle between these projected vectors.
  • We can compute cos α like so:
    • L’ = L − (cosθi)N (then normalize it)
    • V’ = V − (cosθo)N (then normalize it)
    • cos α = L’ ⋅ V’
Parameters
  • a = max( θi , θo )
  • b = min( θi , θo )
  • We only need sin(a) and tan(b)
    • We don’t need to compute a and b themselves
  • We can get sine using identity: For any θ, sin2θ + cos2θ = 1
  • We can get tangent using identity: tanθ = (sinθ)/(cosθ)
  • That’s the last piece of the puzzle. We’re ready to look at shader code
Code
vec3 orenNayar(float cos_theta_i, float cos_theta_o, float sigma,
    vec3 L, vec3 N, vec3 V, vec3 diffuseColor){
    cos_theta_i = max(0.0,cos_theta_i); cos_theta_o = max(0.0,cos_theta_o);
    float sin_theta_i = sqrt(1.0 - cos_theta_i*cos_theta_i);
    float sin_theta_o = sqrt(1.0 - cos_theta_o*cos_theta_o);
    float sin_a, tan_b;
    if( cos_theta_i < cos_theta_o ){
        sin_a = sin_theta_i;
        //add small value to prevent division by zero
        tan_b = sin_theta_o / (cos_theta_o + 2E-38);
    }
    else{
        sin_a = sin_theta_o; tan_b = sin_theta_i / (cos_theta_i + 2E-38);
    }
    float sigma2 = sigma*sigma;
    float A = 1.0 - 0.5*sigma2 / (sigma2+0.33);
    float B = 0.45 * sigma2 / (sigma2+0.09);
    vec3 Lprime = normalize( L - cos_theta_i*N);
    vec3 Vprime = normalize( V - cos_theta_o*N );
    float cosine_alpha = max( 0.0, dot(Lprime, Vprime));
    return cos_theta_i * ( A + B * cosine_alpha * sin_a * tan_b) * diffuseColor;
}
Results
  • Lambert (left) vs. Oren-Nayar (right): Notice how Oren-Nayar gives a more “clay” or “terra cotta” appearance
Minnaert
  • In the 1940’s, Minnaert developed this model for lunar dust
  • Only applies to diffuse (but gives a shiny specular-light appearance)
  • Works best with dark base color
  • m = ((N⋅V)(N⋅L))σ − 1
    • sigma is usually between -2 and 2
  • Then multiply m by the baseColor. That’s the diffuse component.
vec3 minnaert( float cos_theta_o, float cos_theta_i, float sigma, vec3 baseColor ){
    return pow( cos_theta_o * cos_theta_i, sigma-1.0 ) * baseColor;
}

Cook-Torrance
  • One more lighting model: One of the earliest PBR ones…
  • In the 80’s, two researchers (Cook & Torrance) developed a different physically based model for specular
  • This uses much more math, however…
  • Accounts only for specular component
Cook-Torrance
(MFA)/(π(cosθo)(cosθi))
  • F = Fresnel factor
  • M = Percent of microfacets that reflect light to viewer
  • A = Attenuation: Self-shadowing percentage due to microfacet effects
  • π = 3.14159…
Fresnel
  • Cook and Torrance gave this for the Fresnel factor:
F = (1)/(2)⎛⎝(f1)/(f2)⎞⎠2⎛⎝1 + ⎛⎝(f3)/(f4)⎞⎠2⎞⎠
  • Where:
    • n = ( − 1 − √(k))/(√(k) − 1)
    • q = √((cosφ)2 − 1 + n2)
    • f1 = q − cosφ
    • f2 = q + cosφ
    • f3 = f2(cosφ) − 1
    • f4 = f1(cosφ) + 1
Code
  • In code:
vec3 cookTorranceFresnel(float cos_phi, vec3 specularColor ){
    //prevent zero denominator
    vec3 k = min( specularColor, vec3(0.99));
    vec3 sqrtk = sqrt(k);
    vec3 n = (vec3(-1.0) - sqrtk) / (sqrtk - vec3(1.0));
    float q = sqrt(cos_phi*cos_phi - 1.0 + n*n);
    float f1 = q - cos_phi;
    float f2 = q + cos_phi;
    float f3 = (f2 * cos_phi)-vec3(1.0);
    float f4 = (f1 * cos_phi)+vec3(1.0);
    float Q1 = f1/f2;
    Q1*=Q1;
    float Q2 = f3/f4;
    Q2*=Q2;
    return 0.5*Q1*(1+Q2);
}
A
  • Cook and Torrance showed:
A = (2(cos ψ)min(cos θo,  cos θi))/(cos φ)
  • Clamp A to 0…1 range
M
  • One common microfacet distribution equation: Assuming a Gaussian distribution of microfacets:
M = (ret)/(4cos4ψ)
  • Where
t = ρ⋅⎛⎝1 − (1)/(cos2ψ)⎞⎠
  • ρ measures surface roughness
Cook-Torrance
  • Finally, we can put it all together…
vec3 cookTorrance(float roughness, vec3 F, float cos_theta_i,
                  float cos_theta_o, float cos_psi, float cos_phi)
{
    float A = 2.0 * cos_psi * min( cos_theta_o, cos_theta_i)/cos_phi;
    A = clamp(A,0.0,1.0);
    float cos_psi_squared = cos_psi*cos_psi;
    float t = roughness*(1.0-1.0/cos_psi_squared);
    float M = roughness*exp(t)/(4.0*cos_psi_squared*cos_psi_squared);
    return vec3(M)*F*vec3(A)/(cos_theta_o*cos_theta_i*vec3(3.14159265358979323));
}
